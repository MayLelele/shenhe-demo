# 工单字段与校验标准

## 1. 当前 `validate_case_snapshot` 必填项（代码）

- **用户备注**
- **提币金额**
- **充值金额**

工文本建议采用多行、半结构格式，每行 `键:值` 或 `键：值`，例如：

```text
用户ID：U123
用户备注：kyc2提币审
提币金额：100
充值金额：200
```

解析逻辑见 `audit_agent/tools.py` 中 `normalize_case_json`。

## 2. 扩展字段（已支持解析，但非“必填”）

- **用户 ID**：在审核台/粘贴文本中写出 `用户ID：` 时，会进入 `parsed_fields`，便于证据链与人工对账；是否纳入「必填」由**业务**决定，修改方式：在 `validate_case_snapshot` 的 `required` 列表中增删并同步本文件。

## 3. 校验通过的含义

- `ready_for_branch == true` 表示**字段齐全、可进分支判断**（就演示规则而言），**不等于**业务上**必然通过**。
- 缺失字段时 `missing_required` 列出未出现的键，Agent 会倾向「信息不足需补充」类结论。

## 4. 补充新字段时的清单

- [ ] 在本文档与（若需）`entries/` 中说明**含义、来源系统、是否必填**
- [ ] 若必填：改 `validate_case_snapshot` 与单元/手工用例
- [ ] 若进规则匹配：在 `01-` / `lookup_policy_rule` 关键词中体现
