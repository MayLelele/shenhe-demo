# 系统能力映射（知识 ↔ 实现）

便于排查「为什么 Agent 说 X」、以及扩展时改哪里。

| 能力 | 说明 | 代码/页面位置 |
|------|------|---------------|
| 规则检索 | 关键词 → 若干条 `POLICY_RULES` 摘要 | `lookup_policy_rule`，数据在 `tools.POLICY_RULES`；完整文字本库 `01-` |
| 时间证据 | 当前 UTC ISO 时间 | `get_audit_utc_time` |
| 字段校验 | 解析 + 缺失字段 + `ready_for_branch` | `validate_case_snapshot` / `normalize_case_json` |
| 会话记忆 | 多轮上下文、SQLite 持久化 | `memory.py`，库文件 `data/memory.db` |
| 反思/质检 | 对初稿第二轮 LLM 修订（真实模式） | `agent._reflect` |
| 证据链 | 用户/工具/助手 顺序 | `agent.collect_evidence_chain`；Mock 会写入 tool 到 memory |
| 审核台 | 浏览器表单、POST JSON | `audit_web.py` + `audit_workbench.html` |
| 判定标签 | 从最终文本抽「倾向*」等 | `audit_web._infer_verdict`（启发式） |

**注意**：知识库本文档**不自动**被 Agent 读入；要对模型生效需通过 `POLICY_RULES` 或外接 RAG/检索服务后再扩展。
