## 1. 背景与范围

- 核心代码在 `audit_agent/`，与 `main.py` 同入口的交互、与 `Agent` 一致的工具、记忆、反思链。

| 文件 | 作用 |
|------|------|
| `audit_agent-overview.html` | 项目结构 + README 化说明 + 各源码全文的静态总览，浏览器直接打开。 |

## 3. 运行与验证命令（摘要）

```bash
cd audit_agent
source .venv/bin/activate   # 若已建虚拟环境
pip install -r requirements.txt
```

- CLI：`python main.py`（真实） / `python main.py --mock`（离线结构演示）。
- 审核台：`python audit_web.py` → 浏览器 `http://127.0.0.1:8765/`。  
- 无 Key 时：表单勾选 **Mock**，或配置 `.env` 后再用真实模式。

排障：若 `Address already in use`，说明默认端口 `8765` 已被占用。可 `lsof -i :8765` 查 PID 后结束进程，或设 `AUDIT_WEB_PORT=8766` 再启。

## 5. 已知限制（复盘时请带上）

- 规则库为**演示片段**，**正式以业务制度与系统数据为准**；Agent 与页面均为辅助，不替代内控签批。
- 反思分支在部分返回下可能以「反思全文」为主展示，**初稿**未必单独出现在同一段里（与 `agent._reflect` 的拼接策略有关）——以终端/Web 中最终 `reply` 文本为准做归档。
- `verdict` 为从文本**启发式**抽取的快捷标签，与法条/内部口径不一致时，以全文结论人工复核。
